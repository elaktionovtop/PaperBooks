# Порядок разработки

## 1. Двухуровневый порядок

### Уровень A — окна (что делает пользователь)

1. Главное окно
2. Окно авторизации
3. Окна редактирования (книги, авторы, пользователи)
4. Окно читателя (поиск, бронирование)

### Уровень B — внутри каждого окна (как всегда)

```
X.0. Сценарии внешнего поведения
X.1. ViewModel
X.2. Services
X.3. Repository
X.4. View
X.5. Внешнее тестирование
```

---

Это **строгий порядок**. Назад не прыгаем.  
**Никакое окно не начинается, пока предыдущее не доведено до X.5**

---

* Проектируем сверху вниз
* Кодируем по сценариям
* VM — центр мира
* View — последняя
* TDD — как костыль, не как религия
